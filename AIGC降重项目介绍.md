# AIGC 降重技能 — 项目介绍

## 项目概述

**AIGC 降重** 是一个专门为中文学术论文设计的 AI 降重引擎，内置于 Claude Code IDE 插件中作为可调用技能。目标是通过系统性的文本改写，降低论文在 PaperYY、PaperPass 等检测器上的 AIGC 疑似率，使机器生成文本接近人类写作风格，同时保留原意和学术严谨性。

核心能力：
- **论文类型自动识别**：通过信号词统计自动判定理工科 / 社科 / 文科，加载对应领域模板
- **三级降重深度**：L1 词汇替换 → L2 句式拆解 → L3 结构性重写，逐级深入
- **双检测器覆盖**：针对 PaperYY 和 PaperPass 的检测特征分别优化
- **自动化迭代管线**：配套 Python 脚本实现"改写→自检→加码→再改写"闭环

**技术栈**：Python (docx/regex/NLP 模式匹配)、PaperYY/PaperPass 检测器逆向分析、Claude Code Skill SDK、Git

---

## 背景与挑战

中文学术论文的 AIGC 检测面临独特挑战。不同于英文的 GPTZero 等检测器，中国的 PaperYY、PaperPass 等平台对以下特征极为敏感：

1. **文言单字压缩**（可/尚/已/未/将/需/均 → 人类偏好双字口语词）
2. **套话句式模板**（"随着...的不断发展"、"扮演着重要的角色"）
3. **学术写作节奏**（每句一个引用的均匀节奏、三段式结构标记）
4. **政策公文腔**（"持续推进"、"系统推进"、"突出抓好"）
5. **方法论流水账**（"采用...对...进行..."链条）

传统的"同义词替换"策略在降低 AIGC 疑似率上效果有限（仅能降 5-10%），需要深入到句式结构和段落节奏层面改写。

## 核心成果

1. **将一篇 70%+ AI 疑似率的论文持续降到 30% 以下**，跨越 PaperPass 和 PaperYY 两个不同检测平台

2. **构建了完整的降重规则体系**：
   - 20 对古典→口语映射规则
   - 14+8 类社科句式模板拆解规则
   - 76 组理工科对照例句库
   - 16 项自检清单（覆盖词汇/句式/结构/统计/引用维度）

3. **开发了自动化迭代工具链**：
   - `l4_check.py`：16 项 AIGC 自检引擎，可独立运行于任意 docx 文档
   - `l5_iterative.py`：T1/T2/T3/Adaptive 四级迭代引擎，支持自适应收敛
   - 规则库覆盖中文论文全章节类型（摘要/引言/方法/结果/结论/致谢）

4. **系统性文档建设**：将零散的经验归纳为 3 份领域专用规则文档（总计 ~120KB，~900 条规则），涵盖理工科和社科两大领域

## 关键能力展示

### 逆向工程能力
- 对 PaperYY/PaperPass 检测报告进行 HTML 解析，提取高度疑似段落和模式特征
- 通过"改写→检测→分析残留→归纳规则"的反向工程循环，系统识别检测器的敏感特征
- 发现 l4_check 自检工具与 PaperYY 在低分段相关性弱的问题，调整为结构性改写策略

### 系统性规则归纳
- 将检测反馈抽象为可复用的模板规则，而非单篇论文的修补
- 规则按深度分级（T1 词汇/T2 句式/T3 结构/Adaptive 自适应），支持渐进式应用
- 按学科分域（理工科/社科/文科），实现论文类型自动识别和规则自动加载

### 分层改写策略
- **词汇层**：单字→双字、文言→口语、AI 高频词替换
- **句式层**：套话模板拆解、压缩结构舒展、引用动词具体化
- **结构层**：编号列举清零、三阶段演变拆句、密集引用削减
- **节奏层**：打破"每句一引用"均匀节奏、长短句交替、段落首尾变换

### 自动化工程能力
- 将降重从纯手工操作升级为"自动改写→自检→迭代"闭环
- 开发通用化工具脚本，使其可应用于任意中文学术论文
- 编写 Python 正则模式处理中文文本的复杂边界情况（全角/半角、专业术语保护、上下文敏感替换）
